We're in the process of planning what will be included in the next Red Hat Build of Quarkus feature release (codename: Fireball) and the following feature release (codename: something beginning with 'G').

A November release date is currently being targeted for Fireball. This means it will likely be based on Quarkus 2.13.x or 2.14.x.

This release is approaching the final stages, so there shouldn't be many issues in the 'To Do' state and other issues should be close to the 'Implemented' state.

  • 2.y-Fireball.GA: Issues that can be fixed upstream around September or October 2022. In the case of issues not requiring an upstream code change, you will have until around November 2022.
  • x.y-G.GA: Issues that can be fixed for the following RHBQ feature release (merge upstream ~February 2023, or ~March for non-upstream impacting issues)
  • Later.GA: Issues that are valid and complete, but can't be fixed in either of the above timeframes

Thanks for your help with this.

{/beforeIssueTable} {#afterIssueTable}

The states are documented here: Quarkus Product Process | JIRA Workflow.

To summarise the states from an engineering perspective...
To Do: New issues yet to be worked on

Analysis In Progress: The issue is being discussed/planned/analysed before actual development begins
Dev In Progress: Coding on the issue is in progress (Developer moves the issue here when coding has begun)
Implemented: The PR for the code is merged and out of the Developer's hands (Developer moves issue to here when the PR is merged)

Resolved: The issue is available in a release (Productization moves issues from implemented to here during release)
